Créer un plugin de skills pour Claude Desktop

Regroupez plusieurs compétences dans un seul paquet, prêt à partager.

Vous avez suivi le guide principal et vous savez ce qu'est une skill (compétence). Cette page va plus loin : vous allez apprendre à regrouper plusieurs skills dans un seul paquet — un plugin privé — que vous pourrez installer en un clic et partager avec vos collègues, votre équipe ou vos amis. Aucune compétence en programmation n'est requise : tout se fait avec des fichiers texte et un outil de compression ZIP.

Pourquoi regrouper ses skills dans un plugin ?

Une seule skill, c'est une fiche recette. Un plugin, c'est le classeur complet. Voici les avantages concrets :

  • Un seul fichier à installer au lieu de dix skills séparées.
  • Un seul fichier à partager avec votre équipe : tout le monde a les mêmes compétences, au même format.
  • Des skills qui se complètent : vous pouvez créer un ensemble cohérent (par exemple, toutes les tâches administratives d'une association).
  • Des mises à jour simples : vous envoyez un nouveau ZIP, les collègues remplacent l'ancien.

La structure d'un plugin : un dossier, plusieurs sous-dossiers

Un plugin n'est rien de plus qu'un dossier contenant plusieurs sous-dossiers, chaque sous-dossier étant une skill. Voici à quoi ressemble un plugin avec trois skills :

mon-plugin/
├── compte-rendu/
│   ├── SKILL.md
│   └── trame.md
├── email-pro/
│   ├── SKILL.md
│   └── signature.txt
└── note-de-frais/
    ├── SKILL.md
    └── categories.md

C'est tout. Pas de fichier de configuration compliqué, pas de code à écrire. Chaque sous-dossier est une skill autonome avec son fichier SKILL.md (les instructions) et ses fichiers compagnons (modèles, exemples, signatures, etc.).

Règle à retenir

  • Chaque skill doit avoir son propre sous-dossier.
  • Chaque sous-dossier doit contenir un fichier nommé exactement SKILL.md (majuscules obligatoires).
  • Les noms de dossiers : pas d'espaces, pas d'accents, pas de caractères spéciaux. Utilisez des tirets : note-de-frais plutôt que Note de Frais.

Écrire ses skills, pas à pas

Chaque fichier SKILL.md suit le même format simple en deux parties : un en-tête (les informations techniques entre les ---) et les instructions (rédigées en français courant, comme si vous expliquiez la tâche à un collègue).

Partie 1 : l'en-tête

L'en-tête contient trois informations, toujours entre deux lignes de --- :

---
name: compte-rendu-reunion
description: Transforme des notes brutes en compte rendu standard.
  À utiliser dès que je fournis des notes de réunion.
---
ChampQuoi mettreLimite
nameUn identifiant court, sans espaces ni accents64 caractères max
descriptionCe que fait la skill et quand l'utiliser200 caractères max

La description est cruciale : c'est elle que Claude lit pour décider quand charger la skill. Soyez explicite : « À utiliser dès que... », « Quand je demande de... », « Si je fournis... ».

Description vague : « Gère les e-mails. »

Description efficace : « Rédige un e-mail professionnel en mon nom. À utiliser dès que je demande d'écrire un mail, un courriel ou un message professionnel. »

Partie 2 : les instructions

Après le second ---, écrivez vos instructions comme vous brieferiez un assistant humain. Voici un modèle :

---
name: email-pro
description: Rédige un e-mail professionnel en mon nom.
  À utiliser dès que je demande d'écrire un mail ou un courriel.
---

# E-mail professionnel

## Contexte
Je suis [votre prénom], [votre poste] chez [votre entreprise].

## Règles
1. Commence toujours par « Bonjour [prénom du destinataire], ».
2. Ton professionnel mais chaleureux, vouvoiement par défaut.
3. Maximum 15 lignes, sauf si je demande un long message.
4. Termine par la signature contenue dans `signature.txt`.

## Ce qu'il ne faut JAMAIS faire
- Inventer des informations que je n'ai pas fournies.
- Utiliser du jargon technique sauf si le destinataire est technique.
- Tutoyer sans que je le demande explicitement.

Quelques conseils pour bien rédiger les instructions :

  • Soyez concret. « Maximum 15 lignes » est plus utile que « sois concis ».
  • Donnez des exemples. Un avant/après aide Claude à comprendre exactement ce que vous attendez.
  • Listez les interdits. Claude suit mieux les règles négatives (« ne fais jamais... ») que les règles trop vagues.
  • Référencez les fichiers compagnons. Si vous mettez un fichier signature.txt à côté du SKILL.md, nommez-le dans les instructions : « utilise le contenu de signature.txt ».

Les fichiers compagnons

Ce sont les fichiers que vous placez à côté du SKILL.md dans le même dossier. Ils servent de modèles, d'exemples ou de données de référence. Exemples :

  • signature.txt — votre signature d'e-mail
  • trame.md — un modèle de document à remplir
  • categories.md — la liste de vos catégories de dépenses
  • exemple-bon.md — un exemple de résultat réussi pour que Claude comprenne le style

Astuce : demandez à Claude de vous aider

Vous pouvez écrire vos skills directement dans Claude Desktop ! Décrivez ce que vous voulez et demandez : « Rédige le fichier SKILL.md pour cette compétence. » Claude connaît le format et produira un fichier prêt à l'emploi.

Emballer le tout en un fichier ZIP

Une fois vos dossiers de skills prêts, il faut les comprimer en un seul fichier ZIP. Voici comment faire selon votre système.

Sur Windows

  1. Ouvrez l'Explorateur de fichiers et allez dans le dossier qui contient vos sous-dossiers de skills (par exemple mon-plugin).
  2. Sélectionnez tous les sous-dossiers (Ctrl+A).
  3. Clic droit > Compresser dans un fichier ZIP (ou « Envoyer vers > Dossier compressé » sur les anciennes versions).
  4. Renommez le fichier ZIP : mon-plugin.zip.

Sur macOS

  1. Ouvrez le Finder et allez dans le dossier qui contient vos sous-dossiers de skills.
  2. Sélectionnez tous les sous-dossiers (Cmd+A).
  3. Clic droit > Compresser.
  4. Renommez le fichier Archive.zip en mon-plugin.zip.

Point important

  • Les sous-dossiers de skills doivent être à la racine du ZIP, pas dans un dossier intermédiaire. Quand vous ouvrez le ZIP, vous devez voir directement compte-rendu/, email-pro/, etc., et non pas mon-plugin/compte-rendu/.
  • Si vous avez un dossier intermédiaire, entrez dedans, sélectionnez les sous-dossiers, et compressez-les depuis là.

Installer le plugin dans Claude Desktop

Une fois votre fichier ZIP prêt, l'installation se fait en quelques clics :

  1. Ouvrez Claude Desktop.
  2. Allez dans Personnaliser (l'icône en forme de curseurs, en bas à gauche de la fenêtre).
  3. Cliquez sur Compétences (ou « Skills » si votre interface est en anglais).
  4. Cliquez sur Importer ou faites glisser votre fichier ZIP dans la zone d'import.
  5. Claude détecte les skills contenues dans le ZIP et les liste. Activez celles que vous voulez.

Pour vérifier que tout fonctionne, lancez une nouvelle conversation et posez une demande qui devrait déclencher l'une de vos skills. Par exemple, si vous avez une skill « compte-rendu », copiez :

Voici mes notes de réunion d'aujourd'hui :
- Présents : Marie, Paul, Sophie
- On a décidé de reporter le lancement au 15 juillet
- Paul s'occupe de prévenir le client
- Sophie met à jour le planning

Transforme ça en compte rendu.

Si Claude charge la skill, vous verrez son nom apparaître dans la réponse (un petit indicateur en haut du message). Si la skill ne se déclenche pas, revoyez la description dans l'en-tête : c'est presque toujours elle qui est en cause.

Partager son plugin en privé

L'intérêt d'un plugin, c'est de pouvoir le partager. Voici les méthodes les plus simples, de la plus accessible à la plus organisée :

Par e-mail ou messagerie

Envoyez le fichier ZIP par e-mail, WhatsApp, Slack ou tout autre canal habituel. Le destinataire le télécharge et l'importe dans Claude Desktop comme décrit ci-dessus. Simple et efficace pour un petit groupe.

Par un dossier partagé (Drive, OneDrive, Dropbox...)

Placez le ZIP dans un dossier partagé. Avantage : quand vous mettez à jour le plugin, tout le monde peut récupérer la nouvelle version au même endroit. Créez un sous-dossier plugins-claude dans votre espace partagé pour que ce soit facile à retrouver.

Avec un lien de téléchargement

Si vous utilisez Google Drive, Dropbox ou OneDrive, créez un lien de partage vers le fichier ZIP. Envoyez ce lien à vos collègues : un clic pour télécharger, puis import dans Claude Desktop.

Gérer les mises à jour

Quand vous améliorez une skill ou en ajoutez une nouvelle, créez un nouveau ZIP et demandez à vos collègues de supprimer l'ancienne version dans Personnaliser > Compétences avant d'importer la nouvelle. Pensez à noter la date ou un numéro de version dans le nom du fichier : assistant-admin-v2-2026-06.zip.

Exemple complet : un plugin « Assistant administratif »

Voici un plugin prêt à l'emploi avec trois skills. Vous pouvez le reproduire tel quel pour vous entraîner, puis l'adapter à vos besoins.

Structure du plugin

assistant-admin/
├── compte-rendu/
│   └── SKILL.md
├── email-pro/
│   ├── SKILL.md
│   └── signature.txt
└── courrier-officiel/
    └── SKILL.md

Skill 1 : compte-rendu/SKILL.md

---
name: compte-rendu-reunion
description: Transforme des notes brutes en compte rendu de réunion structuré.
  À utiliser dès que je fournis des notes de réunion ou que je demande un CR.
---

# Compte rendu de réunion

## Format obligatoire
1. **En-tête** : date, lieu (si précisé), objet de la réunion.
2. **Participants** : liste à puces, par ordre alphabétique du prénom.
3. **Points abordés** : un paragraphe par sujet, avec les décisions prises.
4. **Actions** : tableau avec trois colonnes — Action | Responsable | Échéance.
5. **Prochaine réunion** : date si elle a été fixée.

## Règles de style
- Ton neutre et factuel.
- Phrases courtes, pas de jargon inutile.
- Une page maximum sauf si la réunion couvre plus de 5 sujets.
- Si une information manque (date, lieu), ne pas l'inventer :
  écrire « [à compléter] ».

Skill 2 : email-pro/SKILL.md

---
name: email-professionnel
description: Rédige un e-mail professionnel en mon nom.
  À utiliser dès que je demande d'écrire un mail, un courriel
  ou un message professionnel.
---

# E-mail professionnel

## Règles
1. Commence par « Bonjour [prénom], » ou « Bonjour, » si le
   destinataire n'est pas nommé.
2. Vouvoiement par défaut, sauf si je précise de tutoyer.
3. Ton professionnel mais chaleureux — ni froid ni familier.
4. Maximum 15 lignes, sauf demande contraire.
5. Termine par la signature contenue dans `signature.txt`.

## Interdits
- Ne jamais inventer de détails (dates, montants, noms).
- Pas de formules creuses : « Je me permets de... »,
  « Suite à notre échange téléphonique... ».

Fichier email-pro/signature.txt

Cordialement,
Marie Dupont
Assistante de direction
Société Exemple
Tel : 01 23 45 67 89

Skill 3 : courrier-officiel/SKILL.md

---
name: courrier-officiel
description: Rédige un courrier formel (lettre officielle) avec mise en page
  standard. À utiliser quand je demande un courrier, une lettre formelle
  ou un recommandé.
---

# Courrier officiel

## Mise en page
1. **Expéditeur** en haut à gauche (nom, adresse).
2. **Destinataire** en haut à droite.
3. **Lieu et date** sous le destinataire.
4. **Objet** en gras, sur une ligne.
5. **Corps** : « Madame, Monsieur, » ou titre approprié.
6. **Formule de politesse** : formelle et classique.
7. **Signature** : prénom, nom.

## Règles
- Vouvoiement obligatoire.
- Ton formel et respectueux.
- Si c'est un recommandé, ajouter « Lettre recommandée avec
  accusé de réception » en haut.
- Si des informations manquent (adresse, numéro de dossier),
  écrire « [à compléter] ».

Ce qu'il faut retenir

  • Chaque skill a son propre dossier avec un SKILL.md.
  • Les fichiers compagnons (signature.txt, etc.) vont dans le même dossier que le SKILL.md qui les utilise.
  • Le tout est compressé en ZIP pour être importé d'un coup.

Bonnes pratiques et erreurs courantes

Les bonnes habitudes

  • Une skill = une tâche. Mieux vaut trois petites skills précises qu'une énorme qui fait tout mal. « Compte rendu » et « e-mail » sont deux skills, pas une seule « bureautique ».
  • Testez chaque skill individuellement avant de les emballer ensemble. Importez la skill seule, posez la demande qui devrait la déclencher, vérifiez que Claude la charge et que le résultat vous convient.
  • Versionnez vos plugins. Ajoutez la date ou un numéro de version dans le nom du ZIP : assistant-admin-v3-2026-06.zip. Ainsi, personne ne confond l'ancienne et la nouvelle version.
  • Documentez pour vos collègues. Ajoutez un petit fichier LISEZ-MOI.txt dans le ZIP qui explique ce que contient le plugin et comment l'installer. Ce fichier n'interfère pas avec les skills.
  • Relisez vos descriptions. Si Claude ne charge pas une skill, c'est presque toujours parce que la description ne correspond pas à ce que vous avez demandé.

Résumé en un coup d'œil

Ce qu'il faut retenir

  • Un plugin = un ZIP contenant plusieurs sous-dossiers, chacun avec son SKILL.md.
  • La description (200 caractères max) décide de quand Claude charge la skill : soignez-la.
  • Testez chaque skill séparément avant de tout regrouper.
  • Partagez le ZIP par e-mail, Drive ou tout autre canal habituel.
